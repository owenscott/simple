Cross-browser Compatibility, Responsive Design, and Frameworks

Last week we looked at using simple CSS to change the appearance of a webpage. This week we're looking at some of the situations where CSS is just a little bit more complicated.

1. Challenges With HTML/CSS

Cross-Browser Compatibility

Unfortunately there is no law that says every browser vendor needs to implement every possible HTML element or CSS property. While HTML and CSS are "standards", ultimately the decision rests with the browser vendors on which features of those standards they will or won't support. In short, no one can force Google, Mozilla, or Apple to modify Chrome, Firefox, or Safari - they have to choose to make the browsers work with the newest HTML and CSS standards.

For instance, under the newest version of HTML several new form inputs have been added, including one for picking colors from a pallete. In theory, using the new input element should be simple: you just pop a <input type="color"> into your webpage and the user gets a beautiful color selection tool. Here is an example:

<html>
  <head>
    <title>HTML Color Input Example</title>
  </head>
  <body>
    <h1>An example form with a color input selector.</h1>
    <form>
      <label for="example">Choose a Color: <input name="example" type="color"></label>
    </form>
  </body>
</html>

Responsive Design

A second huge challenge with building non-trivial HTML/CSS websites is getting a website to work seamlessly on different laptops, desktops, mobile phones, and tablets. In the industry, the art of building a site that works well on any device is typically known as "responsive design". Responsive design is about building a site that is so fluid, it adjusts automatically to any device size.

Check out this site on your laptop. Looks pretty reasonable right? Now try checking it out on your mobile phone (http://bit.ly/1CzAHoL). Not so reasonable. I'm sure we've all had the experience of trying to book a bus ticket or flight on a mobile phone, and facing the challenge of a site's layout which simply doesn't respond to the device.

Now visit this site on your laptop. Looks the same as the last one, I hope. but try it on your mobile phone (http://bit.ly/1AdkuqM). With any luck it should look much better. This is the beauty and simplicity of responsive design.

2. Frameworks

Hopefully by now you've realized that HTML and CSS can be very tough to get right on any non-trivial website. Further, because browsers, devices, and the HTML/CSS standards are always changing, you need to be constantly testing your site in dozens of device-browser permutations. You can see how things can get out-of-hand quickly.

For this reason, many developers use frameworks. Frameworks are pre-made combinations of HTML and CSS that you can use for quickly designing and building sites.

Using a framework is different from writing your own CSS. Typically to use a framework you download a CSS file from the framework's website, and include it with your webpage. Then, instead of writing your own CSS, you simply use classnames provided by that framework. For instance, using the Bootstrap framework, <p class="text-danger">...</p> will automatically style a paragraphs text in red, as a warning to the user. (Which you can use, for instance, when you're trying to flag that they filled out a form incorrectly). Frameworks typically provide a huge number of CSS classes that you can use directly in your HTML, eliminating the need to write CSS yourself.

In this class we will be using the Bootstrap framework. However, if you are curious about what other frameworks exist check out this page for an exhaustive list.

Learn More

However, if you want to do your own deep-dive into writing your own modern CSS and responsive design, here are some resources:
